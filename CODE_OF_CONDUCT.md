# Code of Conduct

## Our Pledge

We want AceClass to be an open, respectful, and technically serious project.
Contributors, maintainers, and community members are expected to participate in
a way that makes collaboration safe and productive for everyone.

## Expected Behavior

- Be respectful and direct.
- Focus criticism on ideas, code, and decisions, not people.
- Assume good intent, but be clear about problems and tradeoffs.
- Welcome different levels of experience.
- Share context when making technical claims or review requests.

## Unacceptable Behavior

- Harassment, intimidation, or discriminatory language
- Personal attacks
- Trolling or intentionally disruptive behavior
- Publishing private information without permission
- Repeatedly derailing technical discussion

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this code of
conduct. They may remove, edit, or reject comments, commits, code, issues, and
other contributions that violate these standards.

## Scope

This code of conduct applies within project spaces and in public
representations of the project, including:

- issues
- pull requests
- discussions
- commit and review comments
- project-linked chat or announcement spaces

## Reporting

If you experience or witness unacceptable behavior, report it privately to the
repository maintainers through a non-public channel available for the project.
If the report is security-related, follow [SECURITY.md](SECURITY.md).

## Enforcement Guidelines

Maintainers may take actions including:

- asking for a change in behavior
- removing harmful content
- temporarily locking a thread
- rejecting a contribution
- temporarily or permanently banning a participant from project spaces
